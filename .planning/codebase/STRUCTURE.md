# Codebase Structure

**Analysis Date:** 2024-04-14

## Directory Layout

```
ytb-superchat/
├── extension/          # Chrome Extension source
│   ├── icons/          # Extension assets
│   ├── settings/       # Options page UI and logic
│   └── sources/        # Platform-specific capture scripts
├── src/                # Streamer dashboard source
│   ├── scripts/        # Node.js build and dev scripts
│   └── site/           # Dashboard frontend logic (ESM)
├── ytb-go/             # Go backend (placeholder/future)
│   ├── cmd/            # Go entry points
│   └── internal/       # Private Go packages (WS, HTTP, Session)
├── .github/            # GitHub Actions workflows
└── .planning/          # GSD planning and mapping docs
```

## Directory Purposes

**`extension/`:**
- Purpose: Contains all files for the Chrome browser extension.
- Contains: `manifest.json`, background worker, and content scripts.
- Key files: `manifest.json`, `service_worker.js`.

**`extension/sources/`:**
- Purpose: Host content scripts injected into streaming platforms.
- Contains: Site-specific scapers (YouTube, Twitch, Kick, etc.) and core bridges.
- Key files: `local-chat-bridge.js`, `youtube.js`, `twitch.js`, `dashboard-relay.js`.

**`src/site/`:**
- Purpose: Web-based dashboard for streamers to monitor chat and superchats.
- Contains: UI modules, data store, and rendering logic.
- Key files: `streamer-app.js`, `streamer-store.js`, `streamer-view.js`.

**`src/scripts/`:**
- Purpose: Build, development, and automation tools.
- Contains: Node.js, Shell, and PowerShell scripts for various environments.
- Key files: `build.mjs`, `serve.mjs`, `dev.sh`.

**`ytb-go/`:**
- Purpose: Backend server component.
- Contains: Go 1.22 project structure. Currently mostly empty boilerplate.
- Key files: `cmd/ytb-go/main.go`, `internal/app/app.go`.

## Key File Locations

**Entry Points:**
- `extension/manifest.json`: Extension entry configuration.
- `src/landing.html`: Entry point for the web dashboard site (renamed to `index.html` during build).
- `ytb-go/cmd/ytb-go/main.go`: Go server entry point.

**Configuration:**
- `extension/manifest.json`: Permissions and content script mappings.
- `package.json`: Project scripts and meta-information.

**Core Logic:**
- `extension/service_worker.js`: Central relay logic for the extension.
- `src/site/streamer-app.js`: Main dashboard application controller.

**Testing:**
- Not detected.

## Naming Conventions

**Files:**
- Kebab-case (mostly): `streamer-app.js`, `local-chat-bridge.js`, `manifest.json`.
- Simple names for sources: `youtube.js`, `twitch.js`.

**Directories:**
- Short, descriptive names: `sources`, `site`, `scripts`, `internal`.

## Where to Add New Code

**New Platform Support:**
- Scraper logic: `extension/sources/[platform].js`.
- Manifest update: `extension/manifest.json` (add new URL match and script).

**New Dashboard Feature:**
- Logic: `src/site/streamer-[feature].js`.
- Integration: Add module import and call in `src/site/streamer-app.js`.

**Backend Endpoint (Future):**
- Handler: `ytb-go/internal/httpapi/handlers_[endpoint].go`.
- Model: `ytb-go/internal/model/[model].go`.

## Special Directories

**`extension/icons/`:**
- Purpose: Static assets for extension branding.
- Committed: Yes.

**`out/` (Ignored):**
- Purpose: Build output directory (generated by `build.mjs`).
- Committed: No.
