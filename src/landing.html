Como funciona na prática

O processo foi desenhado para ser local, leve e direto. Você abre os chats, a extensão captura, e o painel exibe tudo centralizado.

1

Abra os chats popout

YouTube, Twitch e Kick são usados como fonte das mensagens em tempo real.

2

A extensão lê o chat

Os content scripts acompanham o DOM das páginas suportadas e identificam novas mensagens.

3

Dados ficam no navegador

As mensagens são processadas e armazenadas localmente usando o storage da extensão.

4

O painel unifica tudo

A interface web consulta os dados locais e mostra os chats em uma única visualização.

5

Gere overlays

Escolha mensagens específicas e leve esse conteúdo para o OBS ou ferramentas similares.

Sem intermediários. Sem servidor obrigatório.

A proposta do projeto é manter o fluxo o mais direto possível: a extensão captura, o navegador processa e o painel consome os dados localmente.

Isso reduz complexidade, evita dependência de serviços externos e mantém a operação mais leve para o streamer.

Estrutura do projeto

  • content_scripts → captura dos chats nas plataformas
  • service_worker → controle da extensão
  • storage → comunicação entre extensão e interface
  • dashboard → exibição, seleção e controle dos chats

Privacidade e processamento local

O projeto foi desenhado para priorizar simplicidade e controle do usuário sobre os dados acessados durante a transmissão.

Nada é enviado para fora

Nenhum dado precisa ser transmitido para servidores externos para que o painel funcione.

Somente mensagens públicas

A extensão atua em páginas de chat suportadas e lida apenas com o conteúdo público exibido nelas.

Sem coleta pessoal

O foco é exibir mensagens ao vivo, sem coletar informações pessoais privadas do usuário.

Tecnologias utilizadas

JavaScript

Dashboard web

Pontos importantes antes de usar

Algumas limitações fazem parte da proposta técnica atual do projeto.

Quais plataformas são suportadas?

O fluxo descrito hoje considera YouTube, Twitch e Kick por meio dos chats popout.

Pode quebrar com mudanças de layout?

Sim. Como a captura depende da estrutura do DOM das plataformas, alterações visuais podem exigir ajustes.

Usa API oficial?

Não. A proposta é operar localmente, sem depender das APIs oficiais das plataformas para leitura do chat.
